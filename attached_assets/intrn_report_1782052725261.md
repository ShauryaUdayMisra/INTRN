## Issues

### ⚠️ Issue 1 — Replit dev error message showing in production

**Routes affected:** `/internships`, `/profile`, `/company`, `/about`

All four return a 404 that includes:

> *"Did you forget to add the page to the router?"*

This is a Replit scaffold development message. Any user who lands on these URLs — via a typo, an old link, or just guessing `/internships` — sees what looks like a broken, unfinished product. Needs a proper branded 404 page.

---

### ⚠️ Issue 2 — Desktop navigation is effectively hidden

The hamburger menu (Browse Internships / Blog / Help & Support) sits at `left: -304px` off-screen on desktop — it's only toggled by a very small icon in the top-left corner that's easy to miss.

On a 1280px desktop:
- **Browse Internships**, **Blog**, and **Help & Support** are not visible in the navbar
- The only way to reach Blog and Help from the homepage is to scroll to the bottom CTA section, or type the URL directly
- Most desktop visitors will never find the nav

**Suggestion:** Add Blog and Help as visible links in the header on desktop, alongside Sign In / Sign Up.

---

### ⚠️ Issue 3 — Logo on the auth page doesn't navigate home

On `/auth`, the "intrn" logo in the top centre has no click target back to the homepage. Users who want to bail out of the auth flow have to use the `← Back` link or their browser back button. Every other page on the site has a working logo → homepage link.

**Fix:** Wrap the auth page logo in a `<Link to="/">` or `<a href="/">`.

---

### ℹ️ Minor — Copyright year is stale

Footer reads `© 2025 INTRN`. It's 2026. Quick one-line fix.

## What's Working Well

- **Signup form** is clean: first name, last name, email, password, confirm password. Split-panel layout (form + marketing panel) looks polished.
- **Login error handling** works: bad credentials show a red toast "Login failed / Unauthorized" in the bottom-right corner.
- **Internship search** (/search) loads 9 live listings with cards, skill tags, location, hours, and Apply Now buttons. Filters are present.
- **Blog** has real content — multiple articles, category filters, a featured post, and a newsletter subscribe form.
- **Help page** is complete: support email, 24hr response time, FAQ sections for both students and companies, and a Report Issue button.
- **Company landing** (/company-info) is a polished pitch page with clear value props and a "start your journey" CTA.
- **Auth guard** is working — `/dashboard` redirects unauthenticated users to `/auth`.
- **Mobile** (375px) renders correctly — responsive layout, readable text, working CTAs.
